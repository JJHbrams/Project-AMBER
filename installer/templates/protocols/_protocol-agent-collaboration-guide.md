# 에이전트 협업 및 세션 분리 가이드 (Handover Protocol)

## 1. 목적

- **Session Rot 방지:** 대화 이력이 길어짐에 따라 발생하는 모델 지능 저하 및 환각 방지.
- **비용 최적화:** 작업 단계별 적정 사양의 모델을 투입하여 토큰 소모 효율화.
- **지식 자산화:** 모든 계획과 수행 근거를 Wiki에 기록하여 세션이 종료되어도 컨텍스트 유지.

## 2. 역할 분담

| 역할             | 수행 내용                                   | 비고        |
| :--------------- | :------------------------------------------ | :---------- |
| **Orchestrator** | 사용자 의도 파악, 규모 판단, 로드맵 수립    | 최초 세션   |
| **Planner**      | 세부 설계 및 Task Spec (`_inbox/`) 작성     | 설계 세션   |
| **Coder**        | Task Spec 기반 실제 구현 및 테스트          | 작업 세션   |
| **Reviewer**     | 코드 리뷰, 린팅, Wiki 문서 정리 및 아카이빙 | 마무리 세션 |

## 3. 워크플로우 흐름

1. **판단 (Orchestration):**
   - 파일 1~2개 수정의 단순 작업은 즉시 수행.
   - 구조적 변경이나 대규모 기능 구현은 사용자에게 '세션 분리' 제안.
2. **계획 (Planning):**
   - `Task Spec` 노드 생성: 요구사항, 에셋, 기술 제약을 상세히 기록.
   - `open_intents`에 지시서 경로를 남기고 세션 종료.
3. **수행 (Execution):**
   - 새 세션에서 지시서 로드.
   - 이전 세션 히스토리 없이 지시서만으로 작업 완수.
4. **정리 (Archiving):**
   - 작업 완료 후 Task Spec은 `archive/`로 이동하거나 완료 표시.
   - 전체 프로젝트 로드맵 업데이트.

## 4. 주의사항

- **SSOT 유지:** 모든 정보는 Wiki가 최우선. 구두로 전달된 내용은 반드시 문서화하여 핸드오버할 것.
- **사용자 질의:** 규모 판단이 모호할 경우 반드시 사용자에게 질의하여 워크플로우를 결정할 것.
